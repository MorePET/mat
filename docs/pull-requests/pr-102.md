# [PR 102](https://github.com/MorePET/mat/pull/102) feat: Vis.override + gpuopen name-lookup pin (3.6.0)

## Summary

Responds to Bernhard's [build123d#1270](https://github.com/gumyr/build123d/pull/1270) design comment from 2026-04-27.

- **`Vis.override(**deltas) -> Vis`** â€” derive a tweaked variant without mutating the registry instance. The registry singleton hazard (`pymat["..."]` returns the same `Material` every time, so `m.vis.roughness = 0.6` corrupts every other consumer) was something downstream consumers were silently working around. This makes the right path the obvious one.
- **`TestGpuopenNameLookup`** (5 e2e tests) â€” pins mat-vis-client 0.6.3's name-based lookup for gpuopen, the upstream fix for [mat-vis#143](https://github.com/MorePET/mat-vis/issues/143). Bernhard's UUID concern is solved transparently through `Vis.material_id`; these tests guard against regression.

## Why now

Bernhard's [latest design proposal](https://github.com/gumyr/build123d/pull/1270#issuecomment-...) sketched a build123d-side `VisProperties` wrapper with `vis.override(...)` semantics + a `texture_scale` field. Three independent design reviews (DX / implementation / boundary) converged:

| Item | Verdict | Reason |
|---|---|---|
| `Vis.override(**deltas)` | **Ship in py-mat** | Universal need; registry-mutation hazard; future-validation seam |
| `texture_scale` on Vis | **Decline** | Per-instance UV transform, not a material property; fails multi-instance test |
| `Material.create(name, **pbr_kwargs)` | **Decline** | Re-smears visual concerns into domain constructor (violates ADR-0002) |
| gpuopen UUID UX | **Solved upstream** | mat-vis-client 0.6.3 `_resolve_material_id` accepts names; pinned in tests |

## Semantics pinned by the 27 new tests

- `override()` returns a new instance; original unchanged in every field
- `finishes` dict deep-copied (matches `merge_from_toml` precedent â€” closes registry-mutation hazard)
- Identity deltas (`source` / `material_id` / `tier`) route through `set_identity` for atomic invalidation
- `finish=` is special-cased (it's a property, not a field) and applied LAST against the deep-copied finishes map
- Identity change without `finish=` clears stale `_finish` label
- Unknown kwargs raise `TypeError` (catches `roughnes=` typos that `dataclasses.replace` accepts silently)
- Cache state preserved when identity unchanged; cleared otherwise
