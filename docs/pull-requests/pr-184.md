# [PR 184](https://github.com/MorePET/mat/pull/184) feat(policy): data licensing policy + CI license gate (#174)

## Summary

Lands the licensing policy + automated enforcement that turns the `_sources.<key>.license` field (added in #150) from "parsed" into "required for merge."

| File | Purpose |
|---|---|
| `docs/data-policy.md` (new) | Full policy: legal context, allowed licenses, off-limits sources, attribution contract |
| `scripts/check_licenses.py` (new) | stdlib-only CI gate; walks every `_sources` entry, blocks on missing/`unknown`/invalid license |
| `.github/license-ratchet.txt` (new) | Empty placeholder for #175 audit window; self-deletes after audit |
| `.pre-commit-config.yaml` | New `check-licenses` local hook on data/ + script/ + ratchet changes |
| `.github/workflows/ci.yml` | New `check-licenses` job, threaded into `summary.needs` |
| `CONTRIBUTING.md` | Provenance section rewritten ŌĆö `_sources` example with `license`, off-limits summary |
| `LICENSES-DATA.md` (new) | Generated initial state (no CC-BY sources yet) |
| `tests/test_check_licenses.py` (new) | 16 tests for the gate |

## Allowed `license` values

`CC0`, `PD-USGov`, `CC-BY-4.0`, `CC-BY-SA-4.0`, `proprietary-reference-only`. The transitional value `unknown` is parseable but **blocked at merge**.

## Audit-window strategy (per ADR-0003 ┬¦4)

`.github/license-ratchet.txt` ships empty. During #175 (the one-time corpus sweep), entries can be added as work-in-progress markers. The file MUST be deleted as the final step of #175 ŌĆö ratchet rather than feature-flag-on-forever, so we can't accidentally leave the gate disabled.

## Failure UX

Each error names `file:material._sources.key` and the offending value:

```
src/pymat/data/metals.toml:stainless._sources.mechanical.density: license='unknown' ŌĆö must resolve before merge
```

## Risks / regressions considered

1. **Stdlib-only script** ŌĆö runs in pre-commit and CI without extra deps; no version drift surface.
2. **`Path.relative_to(REPO_ROOT)` in tests** ŌĆö handled by `try/except ValueError` fallback to absolute path so tmp_path test corpora work.
3. **Ratchet behavior** ŌĆö `.github/license-ratchet.txt` is read at runtime; `# comments` and blank lines ignored; ships empty.
4. **CI Summary** ŌĆö new job added to `needs` list and `FAILED` checks so a failing license gate blocks the rollup status check.
5. **Pre-commit `language: python` for the local hook** ŌĆö uses isolated interpreter; no need for pymat install.
