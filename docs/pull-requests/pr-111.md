# [PR 111](https://github.com/MorePET/mat/pull/111) feat: 3.8.0 — typed Vis.override kwargs (#108)

## Summary

Closes [#108](https://github.com/MorePET/mat/issues/108). Adds a static-typing layer for \`Vis.override\` so IDEs + type checkers catch typos at edit time.

- **\`VisDeltas\`** — \`TypedDict(total=False)\` enumerating every valid override kwarg
- **\`Vis.override(**deltas: Unpack[VisDeltas]) -> Vis\`** — PEP 692 typed signature, gated under \`TYPE_CHECKING\` so 3.10/3.11 work without a runtime dep change
- **\`pymat.vis.Vis\` + \`FinishEntry\`** — re-exported (Material.with_vis already required them in type hints)
- **\`src/pymat/py.typed\`** — PEP 561 marker so mypy / pyright actually read the annotations

## Verified

- pyright on \`vis.override(roughnes=0.5)\` → \`error: No parameter named "roughnes"\` ✓
- pyright on \`vis.override(source=None)\` → \`Argument of type "None" cannot be assigned to parameter "source" of type "str"\` ✓ (was the lying-type bug caught in review)
- 16 new tests in \`tests/test_vis_deltas.py\`: import surface, TypedDict mirrors Vis fields, \`py.typed\` marker present, runtime round-trip via \`**delta_dict\`, identity \`None\` no-op behavior pinned

## Pre-tag review fixes

Three independent reviews (typing correctness / API surface / fresh-user DX) caught two real issues before tagging:

1. **\`VisDeltas.source\` / \`material_id\` was typed \`str | None\`** but \`override\` routes \`None\` through \`set_identity\` which treats it as no-op. The type was lying about what \`None\` meant. Narrowed to bare \`str\`; passing \`None\` is now a static error.

2. **No \`py.typed\` marker.** Without it, mypy strict mode silently treated the entire package as untyped — half the 3.8.0 value would have evaporated.
