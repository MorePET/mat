# [PR 182](https://github.com/MorePET/mat/pull/182) feat(schema): add _sources provenance table (#150)

## Summary

Adds `[<material>._sources]` TOML support for keyed-by-property-path provenance with `_default` fallback and parent-overlay inheritance ‚Äî the foundation of the schema-expansion milestone.

New module `pymat.sources` defines:
- `Source` (frozen dataclass): `citation`, `kind`, `ref`, `license`, `note`
- Short-alias resolution (`density` ‚Üí `mechanical.density`)
- `parse_sources_table` / `merge_sources` helpers

New `Material` API:
- `mat.source_of("mechanical.density")` ‚Üí `Source | None` (resolves `_default` fallback)
- `mat.cite("density")` ‚Üí BibTeX entry (`@misc` for DOI / Wikidata QID / handbook / vendor)
- `mat.cite()` ‚Üí all sources used by this material, deduplicated by citation key

## Design notes (ADR-0003)

The original issue proposed `mat.density.source` as the accessor. **Rejected** because it would change `mat.density` from `float` to a wrapper object ‚Äî direct break of `core.py:403` (`density / 1000`), `clients/python/pymat-mcp/src/pymat_mcp/tools.py:300` (`compute_mass`), and the `*_qty` Pint quantity layer (~50 call sites). `mat.source_of()` and `mat.cite()` accomplish the same goal additively, with zero changes to existing arithmetic.

License field is parsed but **not validated** in this PR ‚Äî enforcement comes in #174.

## Risks / regressions considered

1. **Underscore keys leaking into `AllProperties`** ‚Äî explicit test `test_sources_dont_leak_into_properties` plus loader-level `if key.startswith("_"): continue`.
2. **Existing TOMLs without `_sources`** ‚Äî fully backwards-compatible; `_sources` defaults to empty dict; no API surface changes for materials that don't use it.
3. **`Material.copy()` propagation of `_sources`** ‚Äî `_sources` lives on `_MaterialInternal` as a dataclass field, so `deepcopy(material)` carries it for free. Verified by `test_child_overlays_parent`.
4. **TOML integrity check** ‚Äî `_walk_unknown_groups` now skips underscore-prefixed keys at material-node level (was implicit pass via heuristic; now explicit).
