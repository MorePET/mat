# [PR 112](https://github.com/MorePET/mat/pull/112) test: eager-import smoke for pymat + mat_vis_client

Closes the test gap that let mat-vis-client 0.5.0's \`StrEnum\` import (3.10-incompatible) escape py-mat's CI in April 2026. \`mat_vis_client.schema\` was never imported by any pymat test on 3.10, so the broken module load surfaced only in build123d's CI (which imports \`mat_vis_client.adapters\` eagerly).

Walks every public submodule of \`pymat\` + \`mat_vis_client\` via \`pkgutil.walk_packages\` and asserts each imports cleanly. Parametrized so \`pytest -v\` shows the module name on each row.

**No release planned** — test-only change. The protection kicks in immediately for PRs on this repo; no user installs benefit from publishing a wheel whose only diff is a new test.

## Scope

| What | Tested? | Why |
|---|---|---|
| Every public \`pymat\` submodule | Yes | 12 modules (lazy-loaded categories included) |
| Every public \`mat_vis_client\` submodule | Yes | 4 modules incl. \`schema\` (the one that broke 3.10) |
| \`pint\` / \`periodictable\` / \`uncertainties\` submodules | No | Mature deps with their own CI; not our coordination cost |
